Add a new file (aka chunk) to this container
Chunk is a single file which is like a vmdk or vhd file. The data of volumes in this container will be stored inside this file depending on the need or user specified protection levels.
Default Only segments which are not protected are created in this chunk file.
Mirror This chunk file can only have segments that are 2x mirror protected.
RAID5 This chunk segments must always be part of a RAID 5 protection group

You can create a chunk which allows either one or a comibination of protection level segments to be created within it.

Compress container data
Compression happens at the logical file level. Every 64k aligned blocks of the file are compressed. If the said 64k data is already deduped, then compression scanner will skip such data segments. It is possible for two files with same content to be deduped after being compressed. If you have a data set that has lot of same 4k blocks, a higher space saving can be achieved by running compression followed by dedupe.

Mount, Clone, Snapshot or Delete volume
Op Image Description Click
Backed Clone In this type of clone, the volume you want to clone is also snapshoted. The volume 'volA' that you want to clone moves one step forward due to this cloning operation as seen in the diagram. This gives you a snapshot version of volA when the clone was created.
Direct Clone Unlike the earlier version, a clone is created volA1 without a snapshot. From this point on volA1 and D can be modifed seperately. You will not have a point in time copy of the volume volA1 when it was cloned
Snapshot This creates a regular snapshot of volA and moves the volume one step ahead.
Delete Deletes all the contents of the volume, but does not remove it from the graph. This may or may not free up space in the container depending on how much of the data in that volume is already deduped
Mount Mounts the volume in Drive R:\. You can browse the contents of this volume from windows explorer
